// ==========================================================
// Comprehensive SaaS Platform — Reference Architecture
// ==========================================================
// Scope: a production-grade multi-tenant SaaS with web + mobile clients,
// gateway, microservices, async pipeline, observability, AI services,
// infrastructure, third-party integrations, and a data plane covering
// transactional, analytical, search, and object storage workloads.
//
// Diagram conventions used below:
//   `>`  forward call / dependency / event publish
//   `<`  reverse call (used to emphasise read paths)
//   `<>` bidirectional / sync subscription
//   `--` weak coupling / observed-only
//   `=>` primary path (e.g. ingress hot path)
//
// Color legend (kept consistent across groups):
//   sky    = clients / edge
//   indigo = gateway + identity
//   purple = product microservices
//   coral  = async + workers + scheduling
//   teal   = data plane
//   green  = object & blob storage
//   amber  = search / vector / analytics
//   red    = AI services
//   slate  = observability
//   blue   = infrastructure / platform
//   orange = third-party integrations
//   yellow = ci/cd & developer tooling
//   lime   = pricing / billing (kept distinct for emphasis)
// ==========================================================
